VisionContext

class VisionContext

Global, thread-local context for vision operations. Provides access to the most recent MatchResult from any ImageFinder, ImageClicker, or ImageWaiter method call, enabling fluent coding patterns while preserving rich match information.

Every find/click/wait call in the vision API automatically updates the last match result for the current thread. This lets you write fluent code like: 


if (ImageClicker.click(button)) {
    MatchResult last = VisionContext.getLastMatch();
    System.out.println("Clicked at " + last.getCenter() + " with confidence " + last.getConfidence());
}

For methods that return multiple results (like findAll), the list of results is also available: 


int count = ImageFinder.findAll(template);
List<MatchResult> results = VisionContext.getLastMatchList();

The context is ThreadLocal, so it is automatically isolated per thread — safe for concurrent bot execution. Use clearLastMatch to reset the context for the current thread (e.g., at the start of a bot action).
